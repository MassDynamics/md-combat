# md_combat

Python translation of R's `sva::ComBat` and `sva::ComBat_seq` for batch effect correction.

## Commands

```bash
# Set up Python env
uv sync --extra dev

# Unit tests (no R needed)
uv run pytest tests/test_combat.py tests/test_combat_seq.py -v

# End-to-end tests — compare Python output to pre-computed R benchmark reference outputs
# (no R needed at test time; Parquet files in tests/expected/ are committed)
uv run pytest tests/test_combat_e2e.py -v

# Skip slow tests (full 64K airway dataset)
uv run pytest -m "not slow" -v

# All tests including slow
uv run pytest -v

# Lint
uv run ruff check src/ tests/

# Format
uv run ruff format src/ tests/

# Coverage
uv run pytest --cov=md_combat --cov-report=term-missing
```

## Benchmark reference outputs (`tests/expected/`)

The e2e tests compare Python output against pre-computed **R benchmark reference**
Parquet files committed in `tests/expected/`.  These are generated by
`scripts/generate_expected.R` and only need to be regenerated when the underlying
datasets or R logic changes.

**Regeneration order — IMPORTANT:**

> If you re-run `scripts/export_datasets.R` (which updates the bundled dataset
> parquet files in `src/md_combat/data/`), you **must** immediately re-run
> `scripts/generate_expected.R` as well.  If you skip this step the benchmark
> references will be out of sync with the bundled data and **all e2e tests will fail**.

```bash
# Step 1 — rebuild bundled datasets (only needed when source R data changes)
Rscript scripts/export_datasets.R

# Step 2 — rebuild benchmark reference outputs (always run after step 1)
Rscript scripts/generate_expected.R
```

Both scripts require R with sva, bladderbatch, Biobase, airway, SummarizedExperiment,
and arrow installed (see R Environment section below).

## R Environment (local dev)

R is only needed to regenerate benchmark reference outputs or run the example notebook.
The e2e tests themselves are pure Python and require no R installation.

### 1 — Install R

**macOS (recommended: use `rig` to manage R versions)**

```bash
# Install rig — requires your sudo password once
brew install --cask rig

# Install the latest R release
rig add release

# Verify
Rscript --version
```

If `brew` is not installed: https://brew.sh

**Alternative (macOS / Windows / Linux):** download the installer directly from
https://cran.r-project.org and follow the prompts. After installation,
`Rscript` should be on your `PATH`.

### 2 — Install required R packages

```bash
# From the repo root — installs sva, bladderbatch, Biobase via Bioconductor
Rscript dependencies.R
```

This takes a few minutes the first time (compiles some packages from source).

### 3 — Verify

```bash
Rscript -e "library(sva); library(bladderbatch); cat('R env OK\n')"
```

## CI/CD

All tests (unit and e2e) are pure Python — no R is required in CI.

```bash
uv sync --extra dev
uv run pytest -m "not slow" -v   # fast suite, runs in seconds
```

To also run the slow tests (full 64K airway), the benchmark reference file
`tests/expected/airway_full_combat_seq.parquet` must be present in the repo.
